## Focused Brief: Improving ASReview-Style Screening on Imbalanced Datasets

Practical implications below are **implementation inferences** synthesized from each source for an ASReview pipeline.

| Area | Source | Practical implications for ASReview implementation |
|---|---|---|
| Active learning setup + imbalance | van de Schoot et al., *Nature Machine Intelligence* (2021): https://www.nature.com/articles/s42256-020-00287-7 | Treat screening as a rare-positive problem from day 1; configure balancing and active querying early. Start with at least one irrelevant label to avoid unstable early decision boundaries. |
| Prior/seed strategy | ASReview “Start a review” docs: https://asreview.readthedocs.io/en/stable/lab/project_create.html | Use known relevant + irrelevant records as prior knowledge before screening. Reuse partially labeled records automatically as priors to accelerate the cold start. |
| Active learning bootstrap | Cormack & Grossman (2015), CAL: https://arxiv.org/abs/1504.06868 | A minimal start (short query/topic/single relevant) can work if relevance feedback is continuous. In ASReview-style loops, keep frequent retraining and reviewer feedback rather than waiting for large labeled batches. |
| Simulation design scale | Teijema et al., large-scale simulation (2025): https://link.springer.com/article/10.1007/s41060-025-00777-0 | Benchmark with many runs and many model combinations; single-run conclusions are fragile. Use 1 relevant + 1 irrelevant as a cold-start baseline, then test larger prior sets. |
| Model choices | Brouwer et al. (2023), ATD study: https://systematicreviewsjournal.biomedcentral.com/articles/10.1186/s13643-023-02257-7 | Use `NB + TF-IDF` as a strong baseline in ASReview simulations, then compare LR/SVM/RF. Evaluate with ATD in addition to WSS@95 to detect “hard-to-find” relevant papers. |
| Feature extraction + parameter effects | Ofori-Boateng et al. (2024), *AI in Medicine*: https://www.sciencedirect.com/science/article/pii/S0933365724002318 | Prioritize TF-IDF experiments before Doc2Vec/S-BERT for abstract screening. Test smaller initial training sets during simulation; they can improve ranking behavior on some datasets. |
| Prior vs model sensitivity | Boetje et al. (2024), elusive records study: https://systematicreviewsjournal.biomedcentral.com/articles/10.1186/s13643-024-02587-0 | Feature extractor choice can matter more than classifier choice for difficult positives. Run model-switch checks late in screening, especially when discovery plateaus. |
| Transformer use under severe imbalance | Ng et al. (2023), pBERT screening: https://systematicreviewsjournal.biomedcentral.com/articles/10.1186/s13643-023-02353-8 | On very low prevalence (~2.5%), transformers may be better as reviewer-assist rather than standalone filters. Use them in dual-review workflows (conflict resolution) instead of full automation. |
| Class imbalance handling | Wallace et al. (2010), BMC Bioinformatics: https://pmc.ncbi.nlm.nih.gov/articles/PMC2824679/ | Imbalance-aware sampling (not naive random undersampling) can reduce false negatives. In ASReview, compare balancer settings under low-prevalence datasets before production use. |
| Threshold metrics for imbalance | Saito & Rehmsmeier (2015), PLOS ONE: https://pmc.ncbi.nlm.nih.gov/articles/PMC4349800/ | Optimize and monitor with PR metrics (AUPRC, precision at recall targets), not ROC alone. Use PR-based checkpoints when deciding screening thresholds and stop decisions. |
| Calibration (theory) | Guo et al. (2017), ICML: https://proceedings.mlr.press/v70/guo17a.html | Calibrate model scores before probability-threshold decisions; raw confidence is often miscalibrated. Temperature scaling is a lightweight post-hoc option for ranking confidence correction. |
| Calibration (implementation) | scikit-learn `CalibratedClassifierCV`: https://scikit-learn.org/stable/modules/generated/sklearn.calibration.CalibratedClassifierCV | For imbalanced data, test `sigmoid` first (often more stable with small calibration sets), then `isotonic` when data is large. Keep calibration CV disjoint from model-fitting folds. |
| Statistical stopping | Callaghan & Müller-Hansen (2020): https://systematicreviewsjournal.biomedcentral.com/articles/10.1186/s13643-020-01521-4 | Use target-recall + confidence stopping with random sampling of unscreened items for defensible termination. This is stronger than purely heuristic “N consecutive irrelevant” rules. |
| Practical stopping heuristic | SAFE procedure (2024): https://systematicreviewsjournal.biomedcentral.com/articles/10.1186/s13643-024-02502-7 | Use composite stopping (key papers found + minimum screened + no recent relevants + effort floor). In ASReview projects, define this rule pre hoc in the protocol. |
| Stopper tooling | ASReview stopper API docs: https://asreview.readthedocs.io/en/stable/technical/reference/asreview.models.stoppers.html | Run sensitivity analyses across stopper types (`NConsecutiveIrrelevant`, `QuantileLabeled`, etc.) in simulation first. Avoid hard-coding one stopper across all topics/datasets. |
| Simulation workflow controls | ASReview simulation CLI docs: https://asreview.readthedocs.io/en/stable/lab/simulation_cli.html | Tune `--n-query` (batch size) and `--n-stop` (stop behavior) explicitly in experiments; defaults can hide tradeoffs between speed and recall risk. |
| Reproducible simulation orchestration | ASReview Makita (Zenodo): https://zenodo.org/records/15720003 | Use Makita to generate reproducible simulation matrices over classifier/feature/balancer/querier combinations. Standardize scripts for multi-run Monte Carlo comparisons. |
| Seed reproducibility in pipelines | ASReview Makita Tool: https://asreview.github.io/asreview-makita/ | Fix `Prior Seed` and `Model Seed` in every benchmark run to separate stochastic variance from real model differences. Store seeds in your protocol and results table. |
| Benchmark data for sparse positives | SYNERGY dataset repo: https://github.com/asreview/synergy-dataset | Use SYNERGY-like prevalence (about 1.67% included) for realistic stress tests. Evaluate early-recall and late-recall behavior separately, not only a single summary metric. |

### Practical default stack (if you need a starting point)
1. `TF-IDF + NB` baseline, `max` or `hybrid max+uncertainty`, imbalance balancer enabled.  
2. Priors: start with known relevant + known irrelevant; run seeded multi-run simulations.  
3. Evaluate with `WSS@95`, `ATD`, and PR metrics; calibrate probabilities before threshold-based actions.  
4. Use composite stopping (SAFE/statistical) validated on simulation before live screening.
